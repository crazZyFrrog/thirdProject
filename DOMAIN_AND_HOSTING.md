# Гайд: домен (reg.ru) + хостинг (Beget / Timeweb)

Пошаговая инструкция для сюрприза заказчику — всё можно сделать без его участия, домен потом передадите ему.

---

## Часть 1. Какой хостинг выбрать?

| | **Beget** | **Timeweb** |
|--|-----------|-------------|
| Простота | Очень простой, много гайдов | Современная панель, тоже просто |
| PHP для формы | ✅ | ✅ |
| SSL бесплатно | ✅ | ✅ |
| Цена | ~200 ₽/мес | ~150–300 ₽/мес |
| FTP | ✅ | ✅ |

**Рекомендация:** **Beget** — чуть привычнее для PHP-лендингов и стабильнее документирован. Timeweb — отличная альтернатива, шаги почти те же.

## Часть 2. Покупка домена на reg.ru

### Шаг 1. Регистрация
1. Откройте [reg.ru](https://www.reg.ru/)
2. Создайте аккаунт (ваш email и телефон)

### Шаг 2. Подбор имени
Примеры для салона:
- `master-lt.ru`
- `krasota-lt.ru`
- `salon-lt-kaluga.ru`

В поиске reg.ru проверьте, свободен ли домен `.ru`.

### Шаг 3. Оформление заказа
1. Добавьте домен в корзину
2. **Владелец домена:**
   - **Вариант А (сюрприз):** ваши ФИО — потом передадите заказчику
   - **Вариант Б (лучше юридически):** ФИО заказчика, если знаете — домен сразу его
3. Оплатите (~200–800 ₽/год)
4. Дождитесь активации (обычно 15 мин – несколько часов)

### Шаг 4. Что НЕ включать на reg.ru
- ❌ Перенаправление / парковку reg.ru
- ❌ «Хостинг reg.ru» параллельно с Beget — только DNS

## Часть 3. Регистрация хостинга Beget

### Шаг 1
1. [beget.com](https://beget.com) → «Виртуальный хостинг»
2. Тариф **Blog** или **Start** (PHP 8.x, ~200 ₽/мес)
3. Зарегистрируйтесь и оплатите

### Шаг 2. Привязка домена
1. Панель Beget → **Домены** → **Добавить домен**
2. Введите купленный домен (`master-lt.ru`)
3. Beget покажет **IP-адрес сервера** (например `185.xxx.xxx.xxx`) — сохраните

### Шаг 3. DNS на reg.ru
1. reg.ru → **Мои домены** → ваш домен → **DNS-серверы и управление зоной**
2. Если стоят NS reg.ru (`ns1.reg.ru`, `ns2.reg.ru`) — откройте **Управление зоной**
3. Добавьте/измените записи:

| Имя | Тип | Значение |
|-----|-----|----------|
| `@` | A | IP сервера Beget |
| `www` | A | IP сервера Beget |

4. Удалите лишние A-записи на другие IP (парковка reg.ru)
5. Сохраните. Пропагация DNS: **1–24 часа** (часто 1–2 часа)

### Шаг 4. SSL
1. Beget → **SSL** → Let's Encrypt для вашего домена
2. Включить для `@` и `www`

## Часть 3б. То же на Timeweb (альтернатива)

> **Важно:** домен с reg.ru добавляется **не** в разделе «Сайты», а в **«Домены и SSL»** (левая колонка панели).  
> Раздел «Сайты» — это папка сайта на сервере. У вас уже есть «Основной сайт» с техническим адресом `cu051770.tw1.ru` — к нему и привяжем ваш домен.

### Шаг 1. Регистрация хостинга Timeweb

1. Откройте [timeweb.com](https://timeweb.com) → **Виртуальный хостинг**
2. Выберите тариф (достаточно базового с PHP 8.x, ~150–300 ₽/мес)
3. Зарегистрируйтесь и оплатите
4. После оплаты откроется панель — как на вашем скриншоте

**Что уже есть по умолчанию:**
- сайт «Основной сайт»;
- технический адрес `cu051770.tw1.ru` — по нему сайт уже доступен до привязки своего домена.

---

### Шаг 2. Добавить домен в Timeweb

1. В **левой колонке** нажмите **«Домены и SSL»** (не «Сайты»!)
2. Откройте вкладку **«Мои домены»**
3. Нажмите **«Добавить домен»** (или **«+ Добавить»**)
4. Введите купленный на reg.ru домен, например: `master-lt.ru`
5. Выберите привязку к существующему сайту: **«Основной сайт»**
6. Нажмите **«Добавить»**

Timeweb зарегистрирует домен в своей панели. Пока DNS на reg.ru не настроен, сайт по вашему домену открываться не будет — это нормально.

**Не путать:**
| Раздел | Для чего |
|--------|----------|
| **Домены и SSL** | добавить домен с reg.ru и включить SSL |
| **Сайты** | посмотреть список сайтов, PHP, папку на сервере |
| **Файловый менеджер** | загрузить файлы проекта |

---

### Шаг 3. Узнать IP сервера Timeweb

IP нужен, чтобы указать его в DNS на reg.ru.

**Способ 1 (основной):**
1. На **главной странице** панели (дашборд) найдите блок **«Доступ по FTP»** или информацию о сервере
2. Скопируйте **IP-адрес** — формат `185.xxx.xxx.xxx`

**Способ 2:**
1. **Сайты** → кликните на **«Основной сайт»** или на `cu051770.tw1.ru`
2. В настройках сайта посмотрите IP сервера (если отображается)

**Способ 3 (проверка):**
- Откройте `http://cu051770.tw1.ru` — если технический адрес работает, хостинг настроен верно, осталось только привязать домен через DNS.

Запишите IP — он понадобится на reg.ru.

---

### Шаг 4. Настроить DNS на reg.ru

Домен куплен на reg.ru → DNS правим **на reg.ru**, не в Timeweb (пока NS-серверы reg.ru).

1. Войдите на [reg.ru](https://www.reg.ru/) → **Мои домены**
2. Кликните по вашему домену
3. Откройте **«DNS-серверы и управление зоной»**
4. Убедитесь, что указаны NS reg.ru:
   - `ns1.reg.ru`
   - `ns2.reg.ru`  
   (если другие NS — см. [инструкцию Timeweb](https://timeweb.com/ru/docs/domeny/resursnye-zapisi-domena-dns-zapisi/nastrojka-dns-u-drugih-registratorov/))

5. Откройте **«Ресурсные записи»** / **«Управление зоной»**
6. Добавьте или измените записи:

| Имя (Subdomain) | Тип | Значение (IP Address) |
|-----------------|-----|------------------------|
| `@` | A | IP сервera Timeweb (из шага 3) |
| `www` | A | тот же IP |

7. **Удалите** лишние A-записи на другие IP (парковка reg.ru, старые хостинги)
8. **Не включайте** на reg.ru «Перенаправление» и «Парковку» — сайт должен открываться с Timeweb
9. Сохраните изменения

**Срок обновления DNS:** обычно 1–3 часа, иногда до 24 часов.

**Проверка:** через пару часов откройте [2ip.ru](https://2ip.ru/) → «Проверка DNS» → введите домен → A-запись должна показать IP Timeweb.

---

### Шаг 5. Включить бесплатный SSL

После того как DNS обновился и домен «дошёл» до Timeweb:

1. **Домены и SSL** → **«Мои домены»**
2. Найдите ваш домен → нажмите **шестерёнку** или **«SSL»**
3. Включите **бесплатный SSL-сертификат** (Let's Encrypt)
4. Включите для:
   - `ваш-домен.ru`
   - `www.ваш-домен.ru`
5. Подождите 5–30 минут — сертификат выпустится автоматически

---

### Шаг 6. Загрузить сайт (если ещё не загружен)

1. **Файловый менеджер** (левая колонка) или FTP
2. Откройте папку сайта «Основной сайт» (часто `public_html` или домен `cu051770.tw1.ru`)
3. Загрузите содержимое папки `dist/` после `npm run build`
4. Убедитесь, что на сервере есть `api/config.php` с Telegram-токенами

Подробнее — **Часть 4** ниже.

| Проверка | Ожидание |
|----------|----------|
| `https://ваш-домен.ru` | открывается лендинг |
| `https://www.ваш-домен.ru` | то же (или редирект на без www) |
| `http://cu051770.tw1.ru` | тоже работает (технический адрес) |

### Частые вопросы (Timeweb + reg.ru)

**Где добавить домен — в «Сайты» или «Домены и SSL»?**  
→ **Домены и SSL** → «Добавить домен». В «Сайтах» домен только отображается после привязки.

**Нужно ли нажимать «Создать сайт»?**  
→ Нет, если уже есть «Основной сайт». Новый сайт создавайте только для второго проекта.

**Домен куплен на reg.ru — переносить его в Timeweb?**  
→ Не обязательно. Достаточно A-записей (шаг 4). Перенос регистрации — отдельная процедура, для сюрприза заказчику не нужна.

**Сайт открывается по `cu051770.tw1.ru`, но не по своему домену**  
→ DNS ещё не обновился или A-запись на reg.ru указана неверно. Проверьте IP на 2ip.ru.

**SSL не включается**  
→ Сначала дождитесь, пока A-запись укажет на Timeweb (шаг 4), затем включайте SSL (шаг 5).

## Часть 4. Загрузка сайта на хостинг

### Подготовка на компьютере

```bash
# 1. Заполните .env (контакты)
copy .env.example .env

# 2. Замените example.ru на ваш домен в:
#    index.html, public/robots.txt, public/sitemap.xml

# 3. config.php уже с Telegram-токенами в public/api/config.php

# 4. Сборка
npm run build
```

### Загрузка через FTP (FileZilla)

1. Beget → **FTP** → данные подключения (хост, логин, пароль)
2. FileZilla → подключиться
3. Открыть папку `public_html` (или `ваш-домен.ru/public_html`)
4. Загрузить **всё содержимое** папки `dist/` (не саму папку dist, а файлы внутри)
5. **Проверить:** на сервере есть `api/config.php` с токенами Telegram

### Структура на сервере

```
public_html/
├── index.html
├── assets/
├── images/
├── api/
│   ├── booking.php
│   └── config.php      ← обязательно!
├── .htaccess
├── robots.txt
└── favicon.svg
```

## Частые ошибки

| Проблема | Решение |
|----------|---------|
| Белый экран | Проверить `.htaccess` и загрузку `assets/` |
| Форма не работает | Есть ли `api/config.php` на сервере |
| Сайт не открывается | Подождать DNS, проверить A-запись |
| Бесконечная загрузка | Не включать перенаправление reg.ru |
| SSL не работает | Включить Let's Encrypt в панели хостинга |
