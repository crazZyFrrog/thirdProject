Все заявки **с сайта** и **из админки** дублируются в два канала:

1. **Telegram** — группа «Записи — Мастерская ЛТ»

---

## Уведомления в Telegram (для мастеров)

Все заявки **с сайта** и **из админки** приходят в группу:

**«Записи — Мастерская ЛТ»**

### Настройка (уже выполнена на сервере)

1. Бот **@salon_lt_bot** добавлен в группу (желательно — администратор группы).
2. В `public_html/api/config.php` на Timeweb:
   - `telegram_bot_token` — токен от @BotFather
   - `telegram_chat_id` — ID группы (с минусом, например `-1003975356754`)
3. В группе должны быть: мастера, администратор салона.

### Дополнительный чат (необязательно)

В `config.php` можно добавить массив — дублировать заявки, например, в личку владельца:

```php
'telegram_chat_ids' => ['-1003975356754', '123456789'],
```

Если указан `telegram_chat_id`, он тоже используется (дубликаты не отправляются дважды в один чат).

### Если перестали приходить сообщения

1. Бот **@salon_lt_bot** в группе и не удалён.
2. Кто-то написал боту **/start** в личке (если бот «молчит»).
3. В `config.php` верные `telegram_bot_token` и `telegram_chat_id`.
4. Тест: запись с сайта или из `/admin/`.
5. **Не перезаписывать** `config.php` при загрузке `dist/` с компьютера.

## Уведомления в VK (для мастеров)

Заявки приходят в **беседу** служебного сообщества **«Записи — Мастерская ЛТ (служебная)»** — тот же текст, что и в Telegram.

### Настройка (на сервере в `public_html/api/config.php`)

```php
'vk_access_token' => 'ключ_доступа_сообщества',  // вкладка «Ключи доступа», право «Сообщения»
'vk_peer_id' => 2000000003,                       // число без кавычек; id беседы
'vk_admin_user_id' => '',                         // пусто, если шлём в беседу
```

**Как узнать `peer_id`:** метод `messages.getConversations` — в ответе у нужного чата поле `peer.id` (например `2000000003` = `2000000000 + chat_id`).

### Участники

В беседу VK должны быть добавлены те же люди, что и в Telegram-группе (мастера, администратор).

### Если перестали приходить сообщения в VK

1. Ключ в `config.php` не истёк (при сомнении — перевыпустить в «Ключи доступа»).
2. Верные `vk_access_token` и `vk_peer_id` (не отрицательный id группы).
3. Беседа есть в списке диалогов **от имени сообщества**.
4. В настройках сообщества **Сообщения** включены; чат-бот не мешает (ошибка 912 — отключить бота или настроить отдельно).
5. **Не включать** Callback API на `example.com` — для уведомлений о записи не нужен.
6. Тест: `messages.send` с тем же `peer_id` или запись с сайта.
7. **Не перезаписывать** `config.php` при деплое.

### Секреты

`vk_access_token` — как пароль: не в чат, не на скриншотах. Хранится только в `config.php` на Timeweb.

## Админка записей

**URL:** https://salonlt.ru/admin/

| Вкладка | Назначение |
|---------|------------|
| Запись по телефону | Клиент позвонил — слот закрыт на сайте, уведомление в Telegram и VK |
| Закрыть время | Блок времени без клиента (обед, личное время) |
| Отпуск | Все дни мастера закрыты в выбранном периоде |
| Список записей | Просмотр и удаление (удаление освобождает слот) |

Пароль задаётся в `api/config.php` → `admin_password`.  
Пароль хранится в браузере до закрытия вкладки.
