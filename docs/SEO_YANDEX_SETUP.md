# SEO и Яндекс: пошаговая настройка (без заказчика)

Отдельный аккаунт Яндекса для салона — **нормальная практика**. При передаче проекта заказчику передайте логин/пароль и добавьте его почту **владельцем** в Метрике и Вебмастере.

---

## Этап 1. Аккаунт Яндекса (вы, ~10 мин)

1. Зарегистрируйте новый Яндекс ID, например: `salonlt.seo@...` (любая почта).
2. Сохраните логин и пароль в менеджере паролей — это «служебный» аккаунт аналитики.
3. При передаче салону: смените пароль вместе с заказчиком или добавьте его Яндекс-почту как владельца.

---

## Этап 2. Яндекс.Метрика (вы, ~15 мин)

1. Откройте [metrika.yandex.ru](https://metrika.yandex.ru) под новым аккаунтом.
2. **Добавить счётчик** → адрес сайта: `https://salonlt.ru`
3. Название: `Мастерская красоты ЛТ`
4. Включите (рекомендуется):
   - Вебвизор
   - Карта кликов
   - Аналитика форм (если есть в интерфейсе)
5. Сохраните счётчик.
6. **Скопируйте номер счётчика** (8 цифр, например `98765432`) — он понадобится для `.env`.

### Цели в интерфейсе Метрики (после деплоя)

В счётчике → **Цели** → добавить JavaScript-события:

| Идентификатор цели | Когда срабатывает |
|--------------------|-------------------|
| `booking_open` | Открыли форму записи |
| `booking_success` | Успешно отправили заявку |
| `vk_click` | Нажали «Написать в ВКонтакте» |

Код на сайте уже отправляет эти события — цели нужно **создать вручную** в панели Метрики с теми же идентификаторами.

---

## Этап 3. Яндекс.Вебмастер (вы, ~10 мин)

1. Откройте [webmaster.yandex.ru](https://webmaster.yandex.ru) под **тем же** аккаунтом.
2. **Добавить сайт** → `https://salonlt.ru`
3. Способ подтверждения: **Мета-тег**.
4. Скопируйте **только значение** `content`, например:
   ```html
   <meta name="yandex-verification" content="a1b2c3d4e5f6g7h8" />
   ```
   В `.env` нужна строка: `VITE_YANDEX_VERIFICATION=a1b2c3d4e5f6g7h8`
5. **Пока не нажимайте «Проверить»** — сначала мы вставим код в сайт и вы задеплоите `dist/`.
6. После деплоя → **Проверить** → статус «Права подтверждены».
7. **Индексирование** → **Файлы Sitemap** → добавить: `https://salonlt.ru/sitemap.xml`
8. **Регион** (если спросит): Россия, Калуга.

---

## Этап 4. Файл `.env` на вашем компьютере (вы, ~2 мин)

В корне проекта создайте `.env` (скопируйте из `.env.example`) и добавьте:

```env
VITE_YANDEX_METRIKA_ID=98765432
VITE_YANDEX_VERIFICATION=a1b2c3d4e5f6g7h8
```

Подставьте **свои** значения из этапов 2 и 3.

Сообщите ассистенту в чате: «ID Метрики и верификация готовы» — или соберите сами:

```bash
npm run build
```

---

## Этап 5. Деплой на Timeweb (вы, ~10 мин)

Загрузите в `public_html` **содержимое** `dist/`:

- `index.html`
- `assets/` (целиком)
- `404.html`
- `.htaccess`
- `robots.txt`, `sitemap.xml`
- `images/` (если меняли, в т.ч. `og.jpg`)

**Не перезаписывайте:** `api/config.php`, `api/data/bookings.db`

---

## Этап 6. Проверка (вы, ~15 мин)

1. **Вебмастер** → нажать «Проверить» для `salonlt.ru`.
2. **Метрика** → «Отчёты» → через 15–30 мин должны появиться визиты (откройте сайт с телефона).
3. **Цели** → откройте форму записи, отправьте тест — проверьте `booking_open` / `booking_success`.
4. **OG-картинка** — вставьте ссылку `https://salonlt.ru/` в VK/Telegram: должно показаться фото салона (`og.jpg`).
5. **404** — откройте `https://salonlt.ru/nesuschestvuyushchaya-stranica` — отдельная страница ошибки.
6. **HTTPS** — включается в панели Timeweb (Let's Encrypt). Редирект в `.htaccess` не используем — на Timeweb это даёт цикл `ERR_TOO_MANY_REDIRECTS`.

Инструменты:
- [validator.schema.org](https://validator.schema.org/) — вставить URL сайта
- [webmaster.yandex.ru](https://webmaster.yandex.ru) → диагностика

---

## Этап 7. Передача заказчику

| Что передать | Как |
|--------------|-----|
| Логин/пароль Яндекс (служебный) | Безопасный канал |
| Ссылка на Метрику | metrika.yandex.ru → счётчик салона |
| Ссылка на Вебмастер | webmaster.yandex.ru |
| Памятка | Этот файл + [HANDOVER.md](HANDOVER.md) |

В Метрике и Вебмастере: **Настройки → Пользователи** → добавить почту заказчика с ролью **Владелец** или **Редактор**.

3. Сохраните как `public/images/og.jpg`
4. `npm run build` и загрузите `images/og.jpg` на хостинг
