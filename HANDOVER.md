## Уведомления в Telegram (для мастеров)

Все заявки **с сайта** и **из админки** приходят в группу:

**«Записи — Мастерская ЛТ»**

### Настройка (уже выполнена на сервере)

1. Бот **@salon_lt_bot** добавлен в группу (желательно — администратор группы).
2. В `public_html/api/config.php` на Timeweb:
   - `telegram_bot_token` — токен от @BotFather
   - `telegram_chat_id` — ID группы (с минусом, например `-1003975356754`)
3. В группе должны быть: мастера, администратор салона.

### Дополнительный чат (необязательно)

В `config.php` можно добавить массив — дублировать заявки, например, в личку владельца:

```php
'telegram_chat_ids' => ['-1003975356754', '123456789'],
```

Если указан `telegram_chat_id`, он тоже используется (дубликаты не отправляются дважды в один чат).

### Если перестали приходить сообщения

1. Бот **@salon_lt_bot** в группе и не удалён.
2. Кто-то написал боту **/start** в личке (если бот «молчит»).
3. В `config.php` верные `telegram_bot_token` и `telegram_chat_id`.
4. Тест: запись с сайта или из `/admin/`.
5. **Не перезаписывать** `config.php` при загрузке `dist/` с компьютера.

## Админка записей

**URL:** https://salonlt.ru/admin/

| Вкладка | Назначение |
|---------|------------|
| Запись по телефону | Клиент позвонил — слот закрыт на сайте, уведомление в Telegram |
| Закрыть время | Блок времени без клиента (обед, личное время) |
| Отпуск | Все дни мастера закрыты в выбранном периоде |
| Список записей | Просмотр и удаление (удаление освобождает слот) |

Пароль задаётся в `api/config.php` → `admin_password`.  
Пароль хранится в браузере до закрытия вкладки.

## Файлы на хостинге (Timeweb)

```
public_html/
├── index.html, assets/, images/, admin/
├── api/
│   ├── config.php          ← НЕ перезаписывать при деплое
│   ├── service-config.json
│   ├── booking.php, admin.php, health.php
│   ├── data/bookings.db    ← НЕ перезаписывать (все записи)
│   └── lib/schedule.php, notify.php, admin-auth.php
```

### Деплой обновлений с компьютера

1. `npm run build`
2. Загрузить **содержимое** `dist/` в `public_html`
3. **Не заменять:** `api/config.php`, `api/data/bookings.db`

| reg.ru | | |
