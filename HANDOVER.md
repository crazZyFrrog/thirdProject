## Админка (отпуска и телефонные записи)

Панель для сотрудников салона: отпуск мастера, запись по телефону, закрытие времени без клиента, просмотр и удаление записей.

### Первый запуск

1. На сервере откройте `public_html/api/config.php` (не перезаписывайте при деплое фронта).
2. Добавьте надёжный пароль:
   ```php
   'admin_password' => 'ваш_секретный_пароль',
   ```
3. Откройте в браузере: **https://salonlt.ru/admin/**
4. Введите пароль и сохраните вкладку в закладках (пароль хранится до закрытия браузера).

### Что умеет админка

| Вкладка | Назначение |
|---------|------------|
| Запись по телефону | Клиент позвонил — слот пропадает на сайте, приходит уведомление в Telegram/VK |
| Закрыть время | Заблокировать интервал без имени клиента (обед, личное время) |
| Отпуск | Закрыть все дни мастера в диапазоне дат |
| Список записей | Последние записи с сайта и из админки; удаление освобождает слот |

### Деплой админки

После обновления проекта загрузите на хостинг:

- `dist/admin/` (папка `admin` из сборки)
- `api/admin.php`
- `api/lib/schedule.php`, `api/lib/admin-auth.php`
- **Не трогать** `api/config.php` и `api/data/bookings.db` на сервере

## Как обновить контакты на сайте

1. На компьютере разработчика открыть проект
2. Отредактировать `.env` (телефон, email, WhatsApp, Instagram)
3. Выполнить `npm run build`
4. Загрузить файлы из `dist/` на хостинг через FTP
5. **Не перезаписывать** `api/config.php` на сервере

См. [CUSTOMIZATION.md](CUSTOMIZATION.md) — правки в файлах `src/components/`, затем сборка и загрузка.

## Если перестали приходить уведомления

### Telegram
1. Проверить, что бот не заблокирован
2. Админ написал боту `/start`
3. Проверить `telegram_bot_token` и `telegram_chat_id` в `api/config.php`
4. Тест: отправить заявку с сайта

### VK
1. Проверить, что сообщество активно
2. Админ **первым** написал сообществу (требование VK API)
3. Перевыпустить access token, если истёк
4. Обновить `vk_access_token` в `api/config.php`

### Форма показывает ошибку
1. Проверить, что файл `api/booking.php` и `api/config.php` на месте
2. Проверить SSL (сайт открывается по HTTPS)
3. Посмотреть логи ошибок PHP в панели хостинга
